# 使用者流程 (User Flow)

> **配套文件**：1-PRD.md（產品需求）、3-TechStack.md（技術選型）
> **更新日期**：2026-05-16

---

## 1. 主要使用者旅程 (Happy Path)

```
[1] 進入頁面
     ↓
[2] 看到左右分欄佈局 + 兩張範例採購單縮圖
     ↓
[3] 拖曳上傳 / 點選檔案 / 點擊範例
     ↓
[4] 左側顯示圖片預覽，按鈕變為 enabled
     ↓
[5] 點擊「一鍵 AI 萃取與驗證」
     ↓
[6] 右側顯示讀取動畫（含跑馬燈狀態文字）
     ↓
[7] AI 解析 + schema 驗證完成
     ↓
[8] 右側展示語法高亮 JSON + meta 資訊列（provider / model / 延遲 / tokens）
     ↓
[9] 使用者可以點「複製 JSON」/「下載 .json」/「重新解析」/「換一張」
```

---

## 2. 畫面狀態 (Screen States)

整個 Demo 是單頁應用，依互動進度切換右側面板狀態：

| 狀態 | 觸發 | 左側 | 右側 |
|------|------|------|------|
| `idle` | 初始載入 | 拖曳區 + 範例縮圖 | 提示文字「上傳採購單以開始」 |
| `previewing` | 檔案就緒 | 圖片預覽 + 「移除」按鈕 + 主要按鈕 enabled | 同 idle（等待觸發）|
| `loading` | 點擊主要按鈕 | 圖片預覽（dim 50%）| 讀取動畫 + 跑馬燈狀態（見 §3）|
| `success` | 驗證通過 | 圖片預覽 | JSON 區塊 + meta 列 + 操作按鈕 |
| `validating` | schema 驗證失敗、re-prompt 中 | 圖片預覽（dim）| 「驗證失敗，正在重試…」訊息 |
| `error` | 兩次都失敗 / API 錯誤 | 圖片預覽 | 紅色錯誤橫幅 + 原始 `content` 折疊區 + `error_code` |

## 3. 讀取動畫的跑馬燈狀態

讓使用者感受到「AI 真的在做事」，依據 backend 階段切換文字：

1. `上傳圖片中…`（client → BFF）
2. `AI 正在解析單據結構…`（BFF → proxy-cli `/api/chat`）
3. `驗證資料格式…`（schema validation）
4. `重新校正中…`（只在 re-prompt 時出現）
5. `完成！`（淡出後切到 success 狀態）

> 實作建議：用 SSE 或單純的 client-side 計時器分階段切換文字（依 4.6s p50 latency 算）。第一版用計時器即可，v2 再做 SSE。

## 4. 錯誤分支 (Error Branches)

| 錯誤情境 | UI 行為 |
|---------|---------|
| 檔案 > 10MB | 上傳前阻擋，toast 提示「圖片過大，請壓縮至 10MB 以下」 |
| 檔案格式不符 (非 png/jpg/webp/pdf) | 阻擋 + toast 提示「僅支援 PNG / JPG / WEBP / PDF」 |
| BFF 連 proxy 失敗 (network) | 紅色橫幅「AI 服務連線失敗，請重試」+ 重試按鈕 |
| `error_code: bad_request` | 紅色橫幅「請求格式有誤」+ 不顯示重試（前端 bug，需 dev 介入） |
| `error_code: auth_invalid` | 紅色橫幅「服務認證失效，請聯絡管理員」 |
| `error_code: quota_exhausted` | 黃色橫幅「AI 服務今日額度已用完，請稍後再試」 |
| `error_code: content_policy` | 黃色橫幅「圖片內容被安全過濾擋下，請換一張圖」 |
| Schema 驗證兩次都失敗 | 紅色橫幅「資料格式驗證失敗」+ 原始 `content` 折疊區（可複製）+ 重試按鈕 |

## 5. 互動微體驗 (Micro-interactions)

- **拖曳進入頁面**：整個拖曳區邊框變色 + 縮放 1.02，提示「放開以上傳」。
- **檔案就緒**：主要按鈕從 disabled grey 轉為主色 + 微脈動動畫，引導點擊。
- **解析完成**：右側 JSON 區塊以「逐行淡入」動畫呈現（增強科技感），總時長 ≤ 600ms 不卡住使用者。
- **meta 資訊列**：顯示 `gemini-2.5-flash · 2.4s · 5,123 in / 412 out tokens`，hover 可看更多細節（actual_source、retry 次數）。
- **複製 JSON**：點擊後按鈕文字變「已複製 ✓」，2 秒後還原。

## 6. Demo 演示腳本 (建議 90 秒)

給銷售/PM 在客戶面前 demo 用：

1. **0-10s**：「企業每天收到上百張這種格式不一的採購單⋯」（指著範例縮圖）
2. **10-20s**：「過去要花業務 5-10 分鐘人工打字、容易出錯。現在⋯」（點擊範例 + 按下萃取）
3. **20-30s**：（等 AI 跑）「我們用最新的 Gemini 2.5 多模態模型，不是傳統 OCR⋯」
4. **30-40s**：（JSON 跳出）「2.4 秒，全部欄位齊全，日期已標準化、數字型別正確⋯」
5. **40-60s**：「重點是這份 JSON 100% 符合資料庫 schema，可以直接寫入 ERP，零人工校對⋯」（指 meta 列強調 tokens / 延遲）
6. **60-90s**：「換一張更亂的⋯」（換範例 2）「⋯結果一樣穩定，這就是 multimodal LLM + 嚴格 schema 驗證的火力。」
